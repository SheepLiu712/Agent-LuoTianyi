我接下来想重写一下数据库功能，包括两个点：

①舍弃对redis服务器的依赖，改为把原来存在redis的内容存在内存中。
- 为每一个用户创建独立的存储对象。
- 需要考虑并发安全，同一个用户不能同时读写，但不同的用户可以同时读写各自的内容。
- 当用户的chat_stream被销毁时才删除redis的内容。


②在写入sql数据库时，由同一个写入器完成，避免多线程写入sql数据库可能带来的问题。

### 要求
只改动src/database部分。暂时不改动上层调用接口，先把底层实现改了再说。也就是说，redis相关的接口还保留，但底层不再使用redis，而是改为内存存储。

### 所需测试
对redis部分
- 测试内存存储功能正常
- 测试并发安全，同一个用户不能同时读写，但不同的用户可以同时读写各自的内容。

对sql数据库部分
- 测试写入器功能正常，能够正确写入数据。
- 测试多线程写入时数据不冲突，能够正确处理并发写入。


### 然后
对所有import Redis的文件进行修改，改为import MemoryStorage，并修改相关代码逻辑以适应新的内存存储实现。
