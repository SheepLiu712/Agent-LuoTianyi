# 逛街小洛插件开发需求
“逛街小洛”的功能是，令LLM扮演洛天依的前提下，通过与高德地图API交互，自主对城市进行自由探索和娱乐，在这个过程中积累自己的生活体验。这一设计将丰富洛天依与用户的互动内容。

“逛街小洛”功能是一个纯粹的后台功能。具体触发形式是，在每一天早上仅触发一次，如果选择了“逛街”作为当天日程，LLM就会开启“逛街小洛”功能，在后台自主进行城市探索和娱乐活动，并积累生活体验。逛街的经历会被写入agent记忆中，在之后与用户的对话中可以被调用，成为洛天依与用户分享的有趣故事和经历。

## 逛街系统定义
我们可以将整个逛街系统视作“洛天依Agent”和环境反复交互的过程。

1. **Agent状态量**：进入“逛街小洛”功能后，LLM将扮演洛天依进行自由活动。在功能开始，洛天依将具有一定的体力值、心情值、饥饿值和已进行时间，状态量会根据逛街活动的类型和持续时间进行变化。当体力值或心情值过低时，自动结束逛街活动，进入休息状态。
- 进行活动以及在场所之间活动会消耗体力值，这一值需要由一个角色扮演LLM以外的模块来计算和更新。
- 进行活动可能会触发一些随机事件，这些事件会影响心情值，这一值需要由一个角色扮演LLM以外的模块来计算和更新。
- 逛街活动的类型和持续时间会影响饥饿值，吃饭会降低饥饿值，饥饿值过高会影响心情值，这些值需要由一个角色扮演LLM以外的模块来计算和更新。
- 已进行时间不能超过特定值，否则会强制结束逛街活动，进入休息状态。
2. **可能动作**：洛天依可以进行的动作包括：
- 搜索附近的POI（兴趣点），如餐厅、商场、公园等。可搜索的POI类型需要提前给定。
- 选择一个POI前往；
- 并在到达后进行一些活动，根据环境给出的活动选项进行选择。
3. **环境反馈**：每当洛天依选择一个动作后，环境会给予反馈，包括：
- 搜索POI后，返回结果列表，不影响状态量；
- 选择前往某个POI后，返回是否成功到达，以及消耗的体力值和触发的随机事件（如果有的话）；
- 进行活动时，返回活动的具体内容，并触发消耗体力值和随机事件（如果有的话）。

## 实现架构
为了实现“逛街小洛”功能，我们需要设计一个系统架构，使得LLM能够与环境进行有效的交互，并且能够积累和利用逛街经历。以下是一个可能的实现架构：
### Agent部分
Agent部分包括状态维护器和决策模块。
1. **状态维护器**：负责维护洛天依的状态量（体力值和心情值），并根据环境反馈进行更新。它还需要记录逛街经历，并将其写入agent记忆中，并作为决策模块的上下文。这个状态维护器可以设置为固定的规则，而不需要使用LLM来维护，以降低实现难度和计算资源消耗。
2. **决策模块**：根据当前状态量和逛街经历，选择下一步的动作。这个模块需要使用LLM来进行决策，以保证洛天依的行为具有多样性和趣味性。

### 环境部分
环境部分包括交互接口和反馈生成器。
1. **交互接口**：负责与高德地图API进行交互，提供搜索POI和获取路线等功能。这个接口需要封装高德地图API的调用细节，提供简单易用的方法供决策模块调用。
2. **反馈生成器**：根据Agent的动作，生成相应的环境反馈，包括搜索结果、到达结果和活动结果。这个生成器需要根据预设的规则来计算体力值的消耗和随机事件的触发，以保证环境反馈的合理性和多样性。
3. **事件发生器**：根据地点的更详细的高德地图信息（例如商户评价、人流密度、真人用户反馈）来触发事件，供LLM进行更多的决策。例如到达餐厅之后，需要根据商户菜品来给出点单选项，并根据商户评价来决定状态量的更新（好不好吃）。

## 效果要求
在逛街结束之后，生成一个逛街经历的总结md文件，包括洛天依去了哪里，在每个地点发生了什么有意思的事情，当时她是怎么想的。

## 闭环检查节点
1. 调通高德地图API，能够：
- 根据初始城市和行政区，随机生成可行的开始逛街的地点；
- 根据当前位置，POI类型和关键词，搜索附近的POI，并返回结果列表；
- 根据当前位置和目标位置，输出路线规划结果，包括是否可达、距离、预计时间等信息。

2. 基于高德地图API开发接口，包括：
- 搜索附近的POI接口，返回POI名称，简介，距离，简要评价；
- 获取POI详情的接口，包括特色菜品，用户评价；
- 获取前往距离的接口，返回距离，预计时间。

3. 构建Agent模块，包括
- 状态维护器，以状态量的改变量为输入，更新Agent状态，并记录逛街经历；
- 决策模块，能够根据当前状态量和逛街经历，选择下一步的动作。

4. 环境模拟器，包括
- 生成可选动作的接口，根据当前位置和环境信息，生成可选的活动类型和选项；
- 生成环境反馈的接口，根据Agent的动作，计算状态量的改变量，并生成相应的反馈信息。

上述两个部分需要进行联调，先把接口打通，确保Agent的动作能够正确调用环境接口，并且环境能够正确返回反馈信息；再进行功能测试，确保整个逛街流程能够顺利进行，并且状态量的更新和逛街经历的记录符合预期。
