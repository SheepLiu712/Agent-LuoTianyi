# 客户端架构
目前，从前端ui到底层通讯，客户端被分为了5层：

1. **UI层**：负责界面展示和用户交互，使用PySide6实现，处理显示以及用户交互事件。
2. **Binder层**：作为UI层和逻辑层之间的桥梁，使用信号和槽结构，将UI事件转换为逻辑层的函数调用，并将逻辑层的结果传回UI层。
3. **信息处理层**：用来处理用户输入的消息，以及服务端返回的agent聊天消息。即尝试将用户输入发送给服务器（发送方式抽象到下一层了），并将服务器返回的消息进行多媒体处理，利用binder层的信号将处理后的消息传回UI层进行展示。
4. **通讯管理层**，即network_client，用于提供与服务器交互的接口，包括登录、注册、发送消息等功能。它将UI层的请求转换为网络请求，并处理服务器的响应。
5. **websocket层**：即ws_transport，负责与服务器建立和维护websocket连接，处理实时通信。
